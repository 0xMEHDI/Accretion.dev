— A 3D top-down Shooter C# — Unity

Currently my most complete game. After working on the platformer listed above, I realized that hand crafting levels wasn't something I found to be very enjoyable. That experience led me down the path of generating levels through code, which I found both challenging and rewarding.

The goal of the game is to survive an increasing number of enemies per wave as they try to overwhelm you with numbers. At the start of each wave, the arena is randomly generated based on a number of preset parameters I setup, chief among which are map size and obstacle percentage. Each individual wave can spawn enemies in varying amounts and at a varying rate, with different health and/or speed values, ensuring every run is unique in both layout and challenge.

This project has taught much about various aspects of game development, from both a programming and design point of view:

  • Programming wise, I've had to tackle collision and enemy AI in a more involved way, as well as navmeshes, event subscriptions, coroutines and some more advanced techniques like the Fisher-Yates shuffle and floodfill algorithms.

  • Design wise, I've had to, for the first time, properly address issues of balancing by adding/removing features to neither trivialize any challenge provided by the game, nor impede the player's ability to properly take control of the character.

— A Survival Colony Sim C# — MonoGame / XNA

This particular game is being actively and heavily worked on as you read this, so I'd like to point out that both this bit of text as well as the video demo above are very much subject to change.

This project is part of a year long assignment for a University module, for which the requirement is to develop a strategy game of some kind. It is being actively worked on by a group of 4 team members including myself.
Our chief inspiration for the game is RimWorld but with a more hand crafted narrative experience.

Working on this has definitely brought about new experiences for me, both positive and negative:

  • I've learned alot from my team members from a technical point of view. Each of us has their own interests and levels of proficiency in different areas of programming, and the exposure to these has led me to put my own ability in perspective and gauge it objectively.

  • I've familiarized myself with the usage of frameworks such as MonoGame, as well as external libraries such as Myra and GeonBit.

  • I've tackled a multitude of lower level features such as pathfinding, lighting, animations, serialization and UI; as well as higher level ones, namely inventory systems, resources and crafting systems.

  • I've also come to learn how to adapt to working within a team where not all team members share the same level of technical ability, drive to create as well designed and fun a game as possible nor the will to make concessions for the sake of the project proceeding smoothly.
